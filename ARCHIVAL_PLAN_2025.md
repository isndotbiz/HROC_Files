## PART 7: DISPOSAL LOG TEMPLATE

### Purpose:
Track documents disposed of according to retention schedule. Required by Document Retention Policy.

### Template Structure:

```csv
Disposal_Date,Document_Type,Document_Title,Original_Date,Retention_Period,Disposal_Method,Authorized_By,Notes
2032-07-27,Policy,Gift Acceptance Policy v1.0,2025-07-30,7 years after superseded,Secure shredding,Jonathan Mallinger,Superseded by v2.0 on 2025-07-27
```

### Fields Explained:
- **Disposal_Date:** Date document was destroyed
- **Document_Type:** Category (Policy, Financial, Corporate, etc.)
- **Document_Title:** Full name of document
- **Original_Date:** When document was created/signed
- **Retention_Period:** Per Document Retention Policy
- **Disposal_Method:** How destroyed (shredding, secure deletion, etc.)
- **Authorized_By:** Person who authorized disposal
- **Notes:** Reason for disposal, superseding document reference

### Disposal Methods:

**Paper Documents:**
- Cross-cut shredding
- Shredding service with certificate of destruction
- Document maintained: Certificate of destruction

**Electronic Documents:**
- Secure deletion (overwrite multiple times)
- Deletion from all backup systems
- Deletion from cloud storage
- Document maintained: Disposal log entry

**Litigation Hold:**
If litigation is anticipated or pending, ALL disposal must stop for relevant documents until hold is lifted.

## PART 8: ARCHIVAL PROCEDURES

### 8.1 When a Policy is Superseded:

1. **Before Creating New Version:**
   - Review current policy in effect
   - Note supersession date
   - Determine 7-year retention end date

2. **When New Version is Adopted:**
   - Board approves new version with resolution
   - New version is signed and dated
   - Old version moved to Archive/superseded_policies/YYYY/MM/

3. **File Naming Convention:**
   - `[Document_Name]_[Date]_superseded_by_[New_Date].pdf`
   - Example: `Financial_Controls_2025-07-30_superseded_by_2026-08-01.pdf`

4. **Create Supersession Note:**
   - Add text file explaining why superseded
   - Reference new version location
   - Note 7-year retention end date

5. **7 Years Later:**
   - Add to disposal log
   - Securely destroy per policy
   - Maintain disposal log entry permanently

### 8.2 When a Resolution is Superseded:

**Note:** Board resolutions are generally PERMANENT records and should NOT be destroyed. However, if a resolution contains errors (like incorrect officer titles) and is re-done:

1. **Keep Both Versions:**
   - Original (even if incorrect) goes to Archive/superseded_resolutions/
   - New corrected version stays in active files
   - Both retained permanently

2. **Document the Correction:**
   - Create Board Resolution explaining the correction
   - Reference both the incorrect and corrected versions
   - Note reason for correction

3. **File Naming:**
   - `Board_Resolution_2025-03_incorrect_titles_superseded_2025-12-15.pdf`
   - `Board_Resolution_2025-03_corrected_2025-12-15.pdf`

### 8.3 Annual Archive Review Process:

**When:** Every December (or end of fiscal year)

**Responsible:** Secretary (currently Jonathan Mallinger)

**Process:**
1. Review all documents created/modified in past year
2. Verify proper categorization in folder structure
3. Check for documents reaching end of retention period
4. Update disposal log for any documents to be destroyed
5. Securely destroy documents per schedule
6. Create annual archive report for board

**Documentation:**
- Checklist of reviewed documents
- Disposal log updates
- Board report summary
- Any issues or recommendations

### 8.4 Document Scanning and Digitization:
